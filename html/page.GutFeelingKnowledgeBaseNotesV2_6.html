Release Notes
- The UniProt proteome accession was chosen as the primary key and the sheet was re-ordered 
- Both RefSeq and Nucleotide accessions from NCBI are listed
- Curation notes about the status of the associated sequences on the various databases were added to the final column
- Additional curation on the metadata about the organisms taken from primary literature was needed, so it has been removed until the next version. 
- While the total number of organisms has been reduced (from 170 in v2.0 to 164 in v2.6), nothing was actually removed. By using the UP accession as the primary key we were able to collapse some orgnisms that had redundant proteomes.

Column Headers
----------------------------------------------------------------
UP ID		UniProt accession for proteome file
UP taxonomy		UniProt taxonomic designation based on the proteome sequence file
UP name		Scientific name based on the proteome sequence file
Assembly ID (UP matched)		NCBI or ENA accession for genome assembly file, matched to the assembly used for the UniProt proteome
RefSeq Accession		RefSeq accession for sequence file, if it exists
GenBank plasmids accessions		Additional NCBI accessions for associated plasmid sequence files
GenBank Accession		NCBI accession for sequence file
GenBank record name		Name of the NCBI sequence record (this is usually the scientific name with additional qualifiers about the sequence)
NCBI taxonomy		Taxonomic ID assigned to the sequence
NCBI taxonomy name		Taxonomic name assigned to the sequence
Curation Notes		any additional notes about the organism's entries in the various databases
